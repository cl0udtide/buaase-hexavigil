# 敌人地形正面推进（terrain-aware front）+ 覆盖预览 — 设计稿

- 日期：2026-06-14
- 分支：`feature/terrain-front-movement`

## 0. 背景与目标

现状：每只怪各自跑 A* 最短路奔核心（`path_service.gd`）；建/拆墙会让全场怪逐个重算（`enemy_manager._on_path_grid_changed`）；运行期完全无视生成期画好的车道（`lanes.gd` 的结果只用于开凿地形，用完即弃）。

后果：所有怪挤在**同一条最短线**上，地形宽窄没有战术意义——一条狭谷和一片开阔平原玩起来一样，玩家只要在那条线上布防即可。地图生成的多样性是视觉的，不是玩法的。

目标：让怪**铺成"一面"推进，吃满地形允许的宽度**。开阔处摊开成宽正面（逼玩家防一整面），狭谷处自然收成一股（天然 choke）。地形形状第一次成为真正的战术变量；玩家用墙塑形地形、把面捏成口。

硬约束（只写一遍）：

- 移动必须是**格子四方向折线**，绝不走斜线。
- **不**用怪间斥力/物理做分散（这是上次流场方案被弃用的原因）。
- 保留并不破坏现有：拆墙（demolisher）模式、核心被围 → 拆墙兜底、飞行直线、footprint 命中、阻挡（block）、Boss 阶段、状态效果、声望结算。

## 1. 核心机制：共享距离场 + 横向相位

把上次流场里**有用的一半**（一张共享的"到核心距离图"）留下，**丢掉**你不要的两样（斥力散开、斜向移动）。

### 1.1 距离场（决定前进方向）

- 从核心做四连通 BFS，覆盖所有可走格，得 `dist[cell]` = 到核心的步数。按 path-mode 各一张：
  - `normal`：绕开地形 + 挡路建筑（墙）。
  - `demolisher`：只绕地形，无视墙。
- 重算触发：`path_grid_changed`（建/拆/修/毁墙、地形变动）。**全场一次 BFS**，取代"每只怪一次 A*"——比现状更省。
- 核心不可达（被墙围死）：`normal` 场到不了核心 → 回退 `demolisher` 场（沿用今天的 `is_core_enclosed_by_path_blockers` 判定）。
- `flying` 不用场：保持现状直线 L 形（`_make_flying_path`）。

### 1.2 局部正面宽度 + 横向相位（让"面"成形）

- 与距离场同批预计算（O(格子)）：每个可走格记录
  - 梯度方向 `g` = `dist` 下降最陡的那个四方向（朝核心；并列时确定性裁决）；
  - 沿 `g` 的**垂直轴**向两侧"走到撞墙/出界为止"的连通可走跨度 = **局部正面宽度 `front_width`**；
  - 本格在该跨度内的位置 `lateral_frac` ∈ [0,1]。
- 每只怪出生时分一个固定 `lateral_phase` p ∈ [0,1)（由出生序号确定性派生，可复现）。
- 推进规则（每帧）：查当前格的 `(g, front_width, lateral_frac)`；目标 = 既沿 `g` 前进、又把 `lateral_frac` 拉向 p 的那个相邻格；四方向 `move_toward` 走一步 → 直角折线。
  - 开阔（`front_width` 大）→ 不同 p 的怪散到不同横向位置 → **铺成一面**。
  - 狭谷（`front_width` 小）→ 所有 p 收敛到同一格 → **收成一股 choke**。
  - 地形宽窄自己决定铺多开，无需怪间交互、无斜线。
- **正面封顶**：p 映射到中心 ± `half_width`，`half_width = f(本波来袭怪量)` 并 clamp 到 `front_width/2`。占位公式 `half_width = clamp(round(K * sqrt(wave_count)), 1, CAP)`（数值随便定，平衡占位）。怪多铺宽、怪少收窄。

## 2. 覆盖预览（替换线预览）

- 现状：`find_path_preview` 每个口给一条线，`map_root_view` 画线。
- 改为**覆盖面**：每个活跃出怪口，算出其怪群将扫过的格子集合 = 从该口沿梯度下行泛洪、按各格 `front_width`/`half_width` 标记的横向覆盖格之并集。**与移动共用同一套场 + 宽度结构**（保证"预览 = 实际会走的范围"，不骗玩家）。
- 渲染：复用现有范围覆盖层 SVG 管线（`gen_overlay_assets` 手写 SVG → `modulate` 贴色，同技能/建筑范围预览的画法）。
  - **每个出怪口用显著不同的颜色**（用户指定，便于读图）；多口覆盖重叠区做叠色。
- 效果：开阔口预览是**一大片**，狭谷口是**一条窄带**；玩家砌墙时能**实时看着覆盖收缩成隘口**。
- API：`find_path_preview` → 返回按口分组的 coverage（`Dictionary{spawn_key: Array[Vector2i]}`）+ 状态（ok / no_path / core_enclosed）。`build_validator` 用到的"砌墙后是否仍有通路"判定改为"`normal` 场是否仍能到核心"。

## 3. 怪绕开拦路干员

- 现状：单位不进路径阻挡集，怪沿线**穿过**干员格；被 `block_count` 接住的怪停下接敌，超出的怪照穿（视觉重叠），不绕。
- 改为：推进时若下一步目标格被某单位占用（`unit_runtime_id ≥ 0`）且本怪**未被该单位阻挡** → 改选另一个同样递减（或同层）的四方向格绕过；开阔有空格则绕流，1 格宽无替代则在前一格排队等待。
- 与 block 协同：单位仍按 `block_count` 接最近的 N 只（`unit_actor` 现有逻辑不动）；绕行只改"没被接住的怪"的走位——一个干员挡不住一整面，但在窄口仍能卡死整股。
- 飞行 / `unblockable` 不变。

## 4. 顺手修：出怪口被迷雾盖住

- 现状（`map_manager.reveal_area` + `map_root_view._draw`）：`reveal_area` 显式跳过 `spawn_key` 格，使出怪口永不 `discovered`（防止玩家从边缘口格白嫖邻接探索）；标记穿透迷雾常显，但格 `discovered=false` → `_draw` 用迷雾瓦片盖住，看不见底下地形。
- 修法：把"视觉可见"与"`discovered`（探索经济邻接）"**解耦**。出怪口格在 `_draw` 里画其真实瓦片 + 地物（未 `discovered` 但 `spawn_key != ""` → 走可见分支，而非迷雾分支），但**不置 `discovered=true`**（探索经济不变式保持）。
- 范围：默认只揭示出怪口格本身，不揭示邻格（守住经济不变式）。

## 5. 受影响组件

- 距离场 + 宽度结构服务：倾向**扩展 `path_service.gd`**（复用其 blocked/mode 逻辑与 `path_grid_changed` 接线），而非新建。
- `enemy_movement_controller.gd`：由"跟随 A* 路径数组"改为"沿场逐步朝相位目标走"；保留 blocked/facing；`crowd_offset` 多半被相位走位取代（保留为兜底）。
- `enemy_manager._on_path_grid_changed`：不再逐怪重算 A*；改为触发场重建（若场服务自身已接 `path_grid_changed`，此处可空）。
- `combat_hud_controller.gd` + `map_root_view.gd`：线预览 → 覆盖面预览 + 每口配色。
- `unit_actor.gd` / movement：绕单位走位。
- `map_manager.gd` + `map_root_view.gd`：出怪口视觉揭示。
- **不动**：demolisher / flying / 核心兜底 / footprint / Boss / 状态 / 声望。

## 6. 数据流

砌墙 → `path_grid_changed` → 场服务重建 `dist` 场 + 宽度结构（按 mode）→ (a) 怪下帧起按新场推进；(b) 覆盖预览重算重绘。
出怪 → 分配 `lateral_phase`。
每帧每怪：查当前格 `(g, front_width, lateral_frac)` → 定目标格 → 绕单位校正 → 四方向 `move_toward` 一步。

## 7. 边界与失败处理

- 无通路（`normal` 场不达核心）：预览 `no_path`；或 `core_enclosed` → demolisher（同今天）。
- 梯度并列、目标格被占、怪被推/击退后：无状态依赖，直接按"当前格的场信息"续走 → 天然鲁棒。
- 怪量为 0 / 单口 / 1 格宽全程：`half_width ≥ 1`，退化成单股（≈今天的线，但仍走折线）。

## 8. 测试

- 纯函数（决定性，沿用 `test_map_generation.gd` 的 `_expect` 风格）：BFS 距离场正确性；`front_width`/`lateral_frac` 标注；相位 → 目标格映射；封顶公式。
- 行为：开阔图怪铺开宽度 ≈ `min(front_width, 2*half_width)`；狭谷收成 1 股；建墙后场/预览更新；绕单位（开阔绕行 / 窄口排队）；出怪口格可见且 `discovered` 仍为 false。
- 复现性：同种子同波 → 怪走位与覆盖逐格一致。

## 9. 不做（YAGNI）

- 不做流体 / 斥力 / 斜向。
- 不做生成期车道的运行期复用（场 + 地形已自带多样性）。
- 不做每怪独立 A*（被共享场取代）。
- 不做小队编组（用户选了"铺成一面"，不是"分小队"）。
