# Boss 特效美术生成：提示词与工作流

> 只管 **Boss 战斗特效**（阶段爆发、范围法术、地面危险区、反弹等）。生成工具 gpt-image-2（网页端，支持参照图），人工生成、AI 实装。
> 特效是序列帧动画表：一张图里横向排 N 帧（代码按 `hframes` / `frame_count` / `fps` 播放，见 `enemy_actor.play_follow_effect` / `spawn_world_effect`），裁成透明 PNG 存 `assets/effects/enemies/<名>_strip.png`。
> 不管角色立绘（`CHARACTER_ASSET_GENERATION_PROMPTS.md`）、地图地块（`MAP_ASSET_GENERATION_PROMPTS.md`）。

## 1. 画风五条

写提示词与验收只看这五条：

1. **同台基准**：与干员/敌人 sprite 同族——干净赛璐璐、2-3 阶硬影界、克制；特效服务于读屏，不糊住角色与战场。
2. **一眼读懂 + 颜色编码**：物理=暖白/橙、法术=青紫、真伤=金白、灼烧=橙红、冰霜=青蓝。玩家一眼认出是哪种威胁。
3. **序列帧节奏**：横向一行等距 N 帧（默认 6 帧），走"出现 → 最盛 → 消散"，末帧接近消失；循环类（地面火）首尾可接。
4. **干净抠图**：纯色 #FF00FF 背景出图，抠成透明；每帧同尺寸、主体居中、不跨帧。
5. **不画**：背景、地块、UI、文字、角色本体、写实火焰照片、霓虹强辉、投到帧外的辉光。

## 2. 工作流

1. **附参照图**：一张现有特效表（`assets/effects/enemies/boss_phase_transition_strip.png`）锚序列帧节奏，一张角色 sprite 锚画风。
2. **生成**：复制 §3 提示词头 + §4 对应块，一次出一张序列帧表，存 `assets/effects/enemies/raw/<名>_source.png`。
3. **裁切入库**：#FF00FF 抠透明，确认每帧等距、同尺寸，整张存 `assets/effects/enemies/<名>_strip.png`（代码按帧数切，不用单独切帧）。
4. **导入提交**：`/Applications/Godot.app/Contents/MacOS/Godot --headless --import --path . --quit`，提交 `.png` 与 `.png.import`。
5. **跑偏就重摇**：颜色编码错、糊住角色、帧不等距、有 #FF00FF 残边——指出违反画风第几条，直接重生成，不后期修补。

## 3. 通用提示词头（每次复制在最前）

```text
请生成一张特效序列帧表：纯色 #FF00FF 背景，横向一行排 6 个等距、等尺寸的方形帧（从左到右），表现同一个特效从【出现 → 最盛 → 消散】的完整过程；最终抠成透明 PNG。

正交俯视、无透视。附图是本游戏的角色 sprite 与现有特效表，风格必须同族：干净赛璐璐、2-3 阶硬影界、克制的发光，不要写实火焰照片、不要霓虹强辉。特效只画能量/火焰/冰晶本身，居中悬空。

禁止：背景、地块、网格、UI 边框/文字/数字、角色或敌人本体、投到帧外的辉光、半透明噪点。
```

## 4. 特效清单（凑凑企鹅·寒怒形态，取材网络梗的冰企鹅 Boss）

凑凑企鹅是冰系 Boss，火雨与爆发统一走**青蓝寒焰 / 冰晶**调，与暖色物理特效区分。

```text
1. coucou_fire_rain_tile（循环）：单格俯视的地面寒焰 patch，青蓝鬼火贴地舔舐、内焰偏白。会平铺到多格并长时间停留，所以首尾帧要能无缝循环、四边色调一致、不带地面底（地块由游戏层）；危险但不刺眼。
2. coucou_corona_pulse：大范围法术普攻（恒星冕风味）。以中心向外快速扩散的一圈青紫法术冲击环，起手一点亮核、随即扩成薄环后消散；中心透空（盖在 Boss 身上）。
3. coucou_reflect_spark：反弹物理的短促反击。被物理击中处迸出的青白冰棱/尖刺反光，4-6 帧极短，干脆利落。
4. coucou_frost_phase_burst：进二阶段的寒霜爆发。青蓝冰晶与寒气向外炸开一圈，比 corona 更厚重、更"碎冰"，作为变身瞬间的强提示。
```

> 通用 Boss 特效（阶段转换光环 `boss_phase_transition_strip`、阶段进入爆发 `boss_phase_enter_area_burst_strip`）已有资产、格式相同；只有某 Boss 需要专属观感时才新做。
> 地面寒焰当前是占位（`ground_hazard_zone.gd` 程序绘制的半透明格）；`coucou_fire_rain_tile` 出图后由代码切换为贴图渲染。

## 5. 入库检查

- 透明、无 #FF00FF 残边；每帧等距、等尺寸、主体不跨帧。
- 颜色编码正确（凑凑系=青蓝寒焰/冰晶）；与干员同框不糊视野、一眼读出威胁类型。
- 游戏内播放：帧数/fps 与调用处一致（默认 6 帧）；循环类（地面火）无跳帧、无接缝。
