# 肉鸽构筑与战斗系统优化方案

> 状态：设计提案（2026-06-10），基于对当前 dev 分支代码与数据的完整调研。
> 文中所有数值均为提案占位值，按"先立预算框架、后统一填数"的原则使用（见 §7）。
> 若实现时机制已有迭代且与本文冲突，以实际代码为准，本文仅作方向参考。

---

## 1. 现状诊断

### 1.1 各系统完成度盘点

| 系统 | 位置 | 现状评价 |
|---|---|---|
| 盟约 | `scripts/combat/covenant_defs.gd` / `covenant_manager.gd` | **唯一有设计脊柱的构筑系统**：人数阈值（2/3 人）、层数、按部署实时结算、与三合一联动。保留并升格为构筑主轴 |
| 遗物 | `data/buffs.json` + `scripts/core/buff_manager.gd` | 约 70 件，实际效果种类仅 ~20 个：大量"同效果 ×1/2/3 稀有度"三连件。抽取为**全池均匀随机**，`rarity` 字段未参与抽取，稀有度轴失效 |
| 随机事件 | `data/events.json` + `scripts/core/random_event_manager.gd` | 死系统：正式地图事件点为 0，效果全部是 ±1~2 资源发放，选项间无决策差异。**内容全删，机制框架（cell 触发、choice 分支）保留复用** |
| 商店+三合一 | `shop_manager.gd` / `operator_progression.gd` | 合格的基础经济。但 6 天 ×5 槽体量下，指定干员二星都靠运气，三星（九合一）不可能发生 → 盟约层数轴空转 |
| 出怪 | `wave_manager.gd` / `night_template_resolver.gd` / `data/wave_templates.json` | 按日定档（1-2 early / 3-4 mid / 5 late / 6 boss），档内随机抽模板、局内不重复。代码支持 `enemy_choices` 加权替换但**无模板使用**。每档仅 4-5 模板，一局基本见完全池 → 局间方差低，玩家无影响力 |
| 敌情预览 | `combat_hud_controller.gd` | **全游戏最好的一块设计**：白天公示今晚构成 + 路线投影 + 悬停摆墙实时重算。后续所有方差系统都应挂在这套公示管线上 |
| 干员技能 | `scripts/combat/skills/` | **全游戏最好的一块内容**：弹药、连锁、牵引、束缚、屏障、反伤、斩杀、伤害类型转换等机制齐备。但构筑层（遗物/事件）与技能层零交互 |
| 敌人 | `data/enemies.json` / `enemy_actor.gd` | 23 种。数值检验轴齐全（高防/高法抗/拆除/飞行/远程/再生/护盾/分裂/死亡爆炸/双形态 Boss）。缺支援型原型（治疗/鼓舞/召唤），且无敌人威胁经济建筑 |

### 1.2 顶层问题（按重要性）

1. **构筑弧线太短**：6 天 = 5 次有效遗物三选一（第 6 晚后的选择是死奖励），却配了一个 70 件的稀释大池。玩家无法"围绕某件遗物成型"。
2. **夜晚体量太小，构筑没有跑道**：一晚 = 一个模板 ≈ 15-25 只怪、刷怪窗口 ~20 秒。再部署 CD（12-70s）一晚轮不到第二次 → 突袭/不屈盟约、彩虹瓮、SP 经济类遗物在多数夜晚无意义。
3. **全局缺"风险—收益"轴**：遗物白送、事件白送、波次固定。玩家全程没有机会用风险换强度，而这是肉鸽策略感的最大来源。
4. **三条经济互不咬合**：行动力+材料 → 建筑；声望 → 干员。两边不抢同一稀缺资源，白天决策退化为两道独立的优化题。
5. **地图——最独特的资产——没有策略身份**：探索只产收入，事件点为 0，刷怪点固定 → 城墙迷宫第 1 天解出后照抄 5 天。
6. **模板抽取是"没有决策的方差"**；Boss 仅 2 个，最终战重复感最强。
7. **平减法防御公式**（`combat_math.gd`：`max(atk-def, 5%atk)`）下，"攻击+X%"遗物是**阈值跨越器**而非线性强度 → 平衡锚点必须是各档敌人的 def/res 阶梯，不能拍脑袋给百分比。

## 2. 总体架构

一句话：**盟约定义流派，商店与三合一是组建流派的过程，遗物是流派的放大器和变形器，事件是唯一的风险与改造渠道，地图探索是情报与机会，夜晚是兑现。**

夜晚的生成公式升级为：

```
夜晚 = 波次模板（构成骨架）
     × 夜晚词缀（系统随机方差，清晨公示）
     × 契约（玩家主动选择的风险）
     × 强度系数（按天数缩放的全局曲线）
```

下文 §3-§6 按系统展开，§7 平衡方法论，§8 落地优先级。

## 3. 构筑侧改造

### 3.1 遗物池重写

**结构调整**

- 合并 1/2/3 稀有度三连件为单件（~70 → ~30 件），稀有度改为**出现时机门控**：
  - 第 1-2 晚：仅普通；
  - 第 3-4 晚：普通 + 稀有；
  - 第 5 晚起：稀有 + 传说；
  - 幕末（见 §6）保底传说。
- 三选一改为**软导向**构成：
  - 槽 A：从玩家已激活/差 1 人激活的盟约对应池中抽；
  - 槽 B：按当前稀有度门控随机；
  - 槽 C：经济类或通用数值类。
- 入口在 `buff_manager.gd::get_random_blessing_choices()`，当前的全池 `pick_random` 替换为分槽抽取。

**职能配比**（重写后 ~30 件的构成）

| 类别 | 占比 | 说明与示例 |
|---|---|---|
| 盟约钥匙件 | ~40% | 每个盟约 2-3 件专属 payoff。例：不屈"复活时眩晕周围敌人 3 秒"；坚守"均摊后每名坚守干员再减伤 8%×坚守人数"；突袭"部署后 3 秒必定闪避且攻击附带短暂停顿"；精准"对生命 50% 以上的敌人伤害 +30%" |
| 机制改变件 | ~25% | 现池中王命凡形、彩虹瓮、小格兰法洛方向的扩充。新增**技能轴词条族**：弹药类技能 +2 发；开启技能时全队 +X SP；屏障类技能效果 +50%；每晚首个技能免费充能；永续类技能攻速加成翻倍 |
| 经济引擎件 | ~20% | 现有声望/商店/建筑系列保留骨架（财眼、锈蚀的铁锤、支援系列） |
| 通用数值件 | ≤15% | 保底安全垫职能，不是池子主体。数值按 §7 断点预算定 |

**空间类新词条**（呼应地图轴）：城墙可承受一次免费拆除；光环建筑范围内干员获得 X；采集建筑兼具 1 阻挡与少量生命。

### 3.2 事件系统：删除重生为"契约系统"

`data/events.json` 现有 10 个事件全部删除。复用 `random_event_manager.gd` 的 cell 触发与 choice 分支机制。

**设计铁律：事件是其它系统都不提供的那种交易——有代价、有改造、有赌。禁止无代价的纯资源发放。**

首批三个原型（P1 落地）：

1. **古代祭坛（改造类）**：为一名干员追加一个盟约 tag（三选一）。盟约构筑的关键补件渠道，文档中预留的接口意愿在此兑现。
2. **黑市商人（契约类）**：核心生命上限 −2，立刻获得一件稀有遗物 / 一名指定盟约干员。
3. **战争赌局（风险类）**：今晚追加 1 条词缀或开启第 4 刷怪点；若核心不漏怪，明早多一次遗物三选一。

后续扩充（P2+）：

- **坍塌隧道（空间类）**：永久封死一个刷怪方向，但其余方向今晚出怪 +50%；
- **废弃哨塔（空间类）**：该格建造费用减半；
- **兑换所（经济桥）**：材料 ↔ 声望双向兑换（汇率劣于产出，作为应急阀）。

**刷新规则**：每天随探索保底生成 1-2 个事件点（探索揭开迷雾时按权重落点），使白天形成"探什么 → 建什么 → 签什么约"三段决策。

### 3.3 商店改进

- **槽位锁定**：每页可锁 1 格未购买干员保留到次日；
- **权重漂移**：随天数推移，出现概率向玩家持有数量最多的 1-2 个盟约轻微倾斜（建议 +15%~25% 相对权重），缓解凑人数的纯随机挫败。

### 3.4 经济桥接

将**魔力矿**升格为跨系统硬通货：高级建筑消耗之外，新增"定向升星"（魔力矿 + 声望直接给指定干员 +1 星，价格高于市场合成期望）与部分事件契约的支付手段。目的：让建筑经济与干员经济在一种稀缺资源上打架。

## 4. 夜晚侧改造

### 4.1 多波结构（最高优先级前置项）

- 每晚 2-3 波，波间 10-15 秒喘息；第 1 天 1 波，随天数递增；
- 目的：让再部署、SP 经济、技能轴、突袭/不屈盟约获得发酵时间；
- 数据层：`wave_templates.json` 的模板改为"波"粒度，夜晚配置引用波列表（或模板内 entries 分组加 `wave_index` + 组间触发条件"上一波清空或超时"）。

### 4.2 夜晚词缀（Night Affix）

每晚随机 0-2 条全局修饰，**清晨随敌情预览一并公示**（公示管线已存在）。词缀独立于模板抽取，与契约可叠加。

首批词缀池（8 条起步）：

| 词缀 | 效果 | 检验的构筑 |
|---|---|---|
| 急行军 | 敌人移速 +30%，生命 −20% | 阻挡/减速 |
| 亡语之夜 | 死亡爆炸/分裂类效果增强 50% | 站位、远程 |
| 空袭警报 | 追加一队飞行单位 | 对空火力 |
| 重甲推进 | 披甲单位防御 +30% | 法术/真伤 |
| 巫术潮汐 | 敌方法抗 +20，我方技能 SP 消耗 −10% | 物理轴、技能轴 |
| 西门失守 | 随机一个刷怪点出怪量 ×2，其余 ×0.5 | 防线弹性 |
| 浓雾 | 远程干员射程 −1，近战攻击 +15% | 配队多样性 |
| 嗜血 | 敌人击杀干员后回血，我方击杀声望 +30% | 风险收益 |

实现：词缀作用于波次展开（数量/构成）或全局 buff 管线（数值），与 `_resolve_wave_entry` / 强度系数同层结算。

### 4.3 模板与方差

- 启用闲置的 `enemy_choices` 加权替换，所有模板补 1-2 个可替换槽位（同一模板每局面貌不同）；
- 每档模板补到 6-8 个（词缀上线后，模板本身的方差压力减半，扩充优先级可后移）。

### 4.4 强度系数

新增按天数缩放的全局曲线（数量与数值分开两条），把难度成长从模板硬编码中解放。平衡期只调曲线，不动模板。建议形态：`第 N 晚需求战力 = 基线 × 1.13^N`（系数待跑批校准，见 §7）。

### 4.5 动态刷怪点

- 第 4-5 天（或第二幕开始）在迷雾区新增 1 个刷怪点，**提前一天在预览中 telegraph**；
- 探索价值落地：提前探到该方向 → 墙和塔可以早建；
- 数据层：模板的 `spawn_key` 从写死 S1/S2/S3 改为"活跃刷怪点槽位"映射，支持数量可变。

### 4.6 敌人扩充

新增原型（每种 1-2 个变体）：

| 原型 | 行为 | 设计目的 |
|---|---|---|
| 战地医师 | 治疗半径内敌人 | 给精准/突袭"斩首"流派存在理由，制造优先级决策 |
| 战鼓手 | 光环加攻速/移速 | 同上 |
| 召唤图腾 | 站桩持续生成小怪 | 必须主动拔点，检验远程/空降 |
| 掠袭者 | **无视核心，直奔最近采集/增益建筑** | 让白天基建必须考虑防守纵深，咬合昼夜两阶段 |
| 疾行者 | 高速低血，绕开重兵路线 | 漏怪检验、防线完整性 |

### 4.7 Boss

- 至少 3 个。现有两个本质都是"双形态数值检验"（一检物理一检法术），第三个直接检验**空间轴**：例如沿途摧毁建筑回血、或分阶段切换进攻路线的 Boss；
- 幕结构（§6）落地后，幕 Boss 在第 3/6 晚提前检验构筑，避免"Boss 内容只在最后一晚"。

## 5. 盟约系统微调

骨架不动，配合上述改动的接口性调整：

- 遗物盟约钥匙件、祭坛加 tag、商店权重漂移均以盟约 id 为索引，`covenant_manager` 暴露查询接口（当前激活档位/层数/差几人）；
- 三合一可行性随天数扩展（§6）后，层数公式才有真实成长空间，届时再做层数收益的平衡 pass；
- 远见（按拥有计算的经济盟约）的"层数 ≥10 解锁卖价折半"在现行体量下不可达，随天数扩展重估或下调阈值。

## 6. 体量扩展：幕结构（3 幕 × 3 天）

**前置条件：§3-§4 的质量改造先行。体量是放大器——放大好决策也放大无聊。** 建议质量改造完成后先在 6 天版本玩测，确认"刚成型就结束"的反馈后再扩。

- **结构**：9 天 = 3 幕 × 3 天，第 3/6/9 晚为幕 Boss；
- **每幕一个难度档 + 专属威胁主题**：第 1 幕基础威胁、第 2 幕支援型敌人登场、第 3 幕拆除与攻城；
- **幕间大节点**：保底稀有/传说遗物 + 新刷怪点 telegraph + 幕间大选择（三选一：定向升星一名干员 / 移除一条词缀 / 一份高级契约）；
- **体量收益**：遗物 8 次三选一；商店 ~45-50 槽使二星稳定可达、三星成为高投入选择，盟约层数轴激活；经济曲线出现中段；
- **时长定位变更需团队拍板**：9 天 ×（白天 1.5-2 分钟 + 夜晚 2-3 分钟）≈ 30-40 分钟/局，偏离文档原定 15-25 分钟。配套缓解：白天操作批量化提效（行动力操作、商店、部署的点击精简）；
- 玩家任意时刻距下一里程碑 ≤2 晚，结构性消灭中段平原。

## 7. 平衡方法论

所有现存数值均为占位（无平衡 pass），填数前先立框架：

### 7.1 锚点：def/res 阶梯

平减法公式下 atk% 是阈值跨越器，先写死各档断点（示例，以现 `enemies.json` 为参考）：

| 档位 | def 区间 | res 区间 | 代表 |
|---|---|---|---|
| early | ≤8 | ≤10 | 士兵、猎狗 |
| mid | 22-36 | 30 | 轻甲卫兵、持盾精锐、术师 |
| late | 36-55 | 35-55 | 重装防御者、萨卡兹大剑手、高阶术师 |
| boss | 55-70 | 40-55 | 两 Boss 一阶段 |

遗物攻击词条与三合一倍率的预算 = "玩家第 N 晚期望攻击应跨越第 N 档断点"。

### 7.2 强度预算

- 基线定义：无遗物、纯 1 星合理配队 → 稳过第 1-2 晚、勉强过第 3 晚、必然过不了第 5 晚；
- 单位预算：每次遗物 pick ≈ 全队有效战力 +10-15%；激活一个 2 人盟约 ≈ 流派内 +25-40%；一次三合一 ≈ 单体 +60%（现 ×1.6 吻合）；
- 词缀 ≈ 当晚难度 +15-25%；契约的奖励预算 ≈ 其难度增量的 1.2-1.5 倍（风险溢价）。

### 7.3 工具

基于现有 `scripts/debug/combat_sandbox.gd` 与 `test_night_template_flow.gd` 搭无头自动战斗跑批：

- 回归矩阵：模板 × 词缀 × 典型构筑（每盟约一套 + 无构筑基线），输出通关余量（核心剩余 HP、超时、击杀率）；
- 验收带：基线队在对应天数模板的通关余量落在目标区间（如第 1 晚余量 60-80%，第 5 晚基线必败、成型构筑余量 20-50%）；
- 每次改数值跑一遍矩阵作为回归保险。

## 8. 落地优先级

| 阶段 | 内容 | 主要触点 | 理由 |
|---|---|---|---|
| **P0** | 夜晚多波结构 | `wave_templates.json`、`wave_manager.gd`、`night_manager.gd` | 不做这条，构筑池一半效果设计了也没意义 |
| **P0** | 夜晚词缀系统（首批 8 条） | 新增 affix 数据 + 结算；预览公示管线复用 | 性价比最高的方差来源，契约与导向都往它身上挂 |
| **P1** | 遗物池重写：合并三连、稀有度门控、分槽软导向 | `buffs.json`、`buff_manager.gd` | 纯数据 + 抽取函数小改，立刻让稀有度轴和构筑感"变真" |
| **P1** | 盟约钥匙件首批（每盟约先 1 件）+ 技能轴词条族首批 | 同上 + 各效果接入点 | 遗物开始引用游戏里的真实机制 |
| **P1** | 事件重生：祭坛 / 黑市 / 赌局三原型，每日保刷 1-2 事件点 | `events.json`、`random_event_manager.gd`、探索落点逻辑 | 风险轴 + 探索价值同时落地 |
| **P2** | 商店锁定槽 + 权重漂移 | `shop_manager.gd` | 凑盟约的摩擦缓解 |
| **P2** | 支援型敌人 + 掠袭者 + 动态刷怪点 | `enemies.json`、`enemy_actor.gd`、刷怪点槽位化 | 优先级决策 + 昼夜咬合 + 防线重构 |
| **P2** | 魔力矿经济桥（定向升星 + 契约支付） | `RunState`、商店/事件 | 白天取舍成立 |
| **P3** | 幕结构 3×3、第三 Boss、模板扩充、`enemy_choices` 启用 | 多处 | 在质量改造玩测确认后扩体量 |

### 风险与开放问题

1. **单局时长定位**（15-25 分钟 → 30-40 分钟）需团队明确拍板，P3 幕结构以此为前提；
2. 坚守 3 人"全队伤害均摊"与不屈复活、王命凡形等组合的极端情形，重写遗物池时需排雷；
3. 词缀与契约叠加的难度上限要在跑批矩阵中单独验收（最坏组合不应无解）；
4. 白天操作提效（批量采集、快捷部署）未展开，幕结构落地前需要单独的 UX pass。
