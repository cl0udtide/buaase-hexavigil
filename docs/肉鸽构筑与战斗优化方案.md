> 文中所有未标注“当前实现”的数值均为提案占位值，按"先立预算框架、后统一填数"的原则使用（见 §7）。

## 1. 现状诊断

### 1.1 各系统完成度盘点

| 系统 | 位置 | 现状评价 |
|---|---|---|
| 盟约 | `scripts/combat/covenant_defs.gd` / `covenant_manager.gd` | **唯一有设计脊柱的构筑系统**：人数阈值（2/3 人）、层数、按部署实时结算、与三合一联动。保留并升格为构筑主轴 |
| 遗物 | `data/buffs.json` + `scripts/core/buff_manager.gd` | 当前 37 件。抽取已改为稀有度门控 + 盟约导向槽 + 随机槽 + 经济/通用保底槽；后续重点不再是“接入 rarity”，而是继续压缩重复数值件、补强机制件和盟约 payoff |
| 随机事件 | `data/events.json` + `scripts/core/random_event_manager.gd` | 已从旧资源点系统改为每日刷新契约事件：开局 2 个，此后每天 1-2 个，活跃上限 4；当前根事件 7 类，包含黑市、赌局、商队、祭坛、佣兵、军械库、裂隙。后续重点是平衡奖励预算和继续扩变体 |
| 商店+三合一 | `shop_manager.gd` / `operator_progression.gd` | 合格的基础经济。但 6 天 ×5 槽体量下，指定干员二星都靠运气，三星（九合一）不可能发生 → 盟约层数轴空转 |
| 出怪 | `wave_manager.gd` / `night_template_resolver.gd` / `data/wave_templates.json` / `data/night_affixes.json` | 已升级为按天数解析整夜多波模板计划，并叠加 0-2 条夜晚词缀；当前 15 个模板、7 条词缀。`enemy_choices` 仍未被模板使用，模板池和 Boss 数量仍偏少 |
| 敌情预览 | `combat_hud_controller.gd` | **全游戏最好的一块设计**：白天公示今晚构成 + 路线投影 + 悬停摆墙实时重算。后续所有方差系统都应挂在这套公示管线上 |
| 干员技能 | `scripts/combat/skills/` | **全游戏最好的一块内容**：弹药、连锁、牵引、束缚、屏障、反伤、斩杀、伤害类型转换等机制齐备。但构筑层（遗物/事件）与技能层零交互 |
| 敌人 | `data/enemies.json` / `enemy_actor.gd` | 23 种。数值检验轴齐全（高防/高法抗/拆除/飞行/远程/再生/护盾/分裂/死亡爆炸/双形态 Boss）。缺支援型原型（治疗/鼓舞/召唤），且无敌人威胁经济建筑 |

### 1.2 顶层问题（按重要性）

1. **构筑弧线太短**：6 天 = 5 次有效遗物三选一（第 6 晚后的选择是死奖励），却配了一个 70 件的稀释大池。玩家无法"围绕某件遗物成型"。
2. **夜晚体量曾经太小，构筑没有跑道**：当前多波结构已经缓解“一晚一个模板”的问题，但第 6 晚仍只有 late+boss 两波，且整体战斗时长与再部署/CD收益仍需要跑批确认。
3. **风险—收益轴已起步但未平衡**：事件和词缀已经提供风险交易，下一步重点是奖励预算、最坏词缀组合和契约叠加难度的跑批校准。
4. **三条经济互不咬合**：行动力+材料 → 建筑；声望 → 干员。两边不抢同一稀缺资源，白天决策退化为两道独立的优化题。
5. **地图策略身份仍需继续强化**：事件点已经每日刷新到探索前沿，但动态刷怪点、建筑受威胁、空间类事件和更强的探索情报价值还未落地。
6. **模板抽取是"没有决策的方差"**；Boss 仅 2 个，最终战重复感最强。
7. **平减法防御公式**（`combat_math.gd`：`max(atk-def, 5%atk)`）下，"攻击+X%"遗物是**阈值跨越器**而非线性强度 → 平衡锚点必须是各档敌人的 def/res 阶梯，不能拍脑袋给百分比。

## 2. 总体架构

一句话：**盟约定义流派，商店与三合一是组建流派的过程，遗物是流派的放大器和变形器，事件是唯一的风险与改造渠道，地图探索是情报与机会，夜晚是兑现。**

夜晚的生成公式升级为：

```
夜晚 = 波次模板（构成骨架）
     × 夜晚词缀（系统随机方差，清晨公示）
     × 契约（玩家主动选择的风险）
     × 强度系数（按天数缩放的全局曲线）
```

下文 §3-§6 按系统展开，§7 平衡方法论，§8 落地优先级。

## 3. 构筑侧改造

### 3.1 遗物池重写

**结构调整**

- 合并 1/2/3 稀有度三连件为单件（~70 → ~30 件），稀有度改为**出现时机门控**：
  - 第 1-2 晚：仅普通；
  - 第 3-4 晚：普通 + 稀有；
  - 第 5 晚起：稀有 + 传说；
  - 幕末（见 §6）保底传说。
- 三选一改为**软导向**构成：
  - 槽 A：从玩家已激活/差 1 人激活的盟约对应池中抽；
  - 槽 B：按当前稀有度门控随机；
  - 槽 C：经济类或通用数值类。
- 入口在 `buff_manager.gd::get_random_blessing_choices()`，当前的全池 `pick_random` 替换为分槽抽取。

**职能配比**（重写后 ~30 件的构成）

| 类别 | 占比 | 说明与示例 |
|---|---|---|
| 盟约钥匙件 | ~40% | 每个盟约 2-3 件专属 payoff。例：不屈"复活时眩晕周围敌人 3 秒"；坚守"均摊后每名坚守干员再减伤 8%×坚守人数"；突袭"部署后 3 秒必定闪避且攻击附带短暂停顿"；精准"对生命 50% 以上的敌人伤害 +30%" |
| 机制改变件 | ~25% | 现池中王命凡形、彩虹瓮、小格兰法洛方向的扩充。新增**技能轴词条族**：弹药类技能 +2 发；开启技能时全队 +X SP；屏障类技能效果 +50%；每晚首个技能免费充能；永续类技能攻速加成翻倍 |
| 经济引擎件 | ~20% | 现有声望/商店/建筑系列保留骨架（财眼、锈蚀的铁锤、支援系列） |
| 通用数值件 | ≤15% | 保底安全垫职能，不是池子主体。数值按 §7 断点预算定 |

**空间类新词条**（呼应地图轴）：城墙可承受一次免费拆除；光环建筑范围内干员获得 X；采集建筑兼具 1 阻挡与少量生命。

### 3.2 事件系统：删除重生为"契约系统"

当前 `data/events.json` 已完成第一轮契约化改造，复用 `random_event_manager.gd` 的 cell 触发与 choice 分支机制。

**设计铁律：事件是其它系统都不提供的那种交易——有代价、有改造、有赌。禁止无代价的纯资源发放。**

当前已落地的原型：

1. **古代祭坛（改造类）**：消耗 2 魔力矿，为一个 `unit_id` 在本局追加盟约 tag；现有与后续同名实例都会继承。
2. **黑市商人（契约类）**：核心生命上限 −2，获得一件稀有遗物。
3. **战争赌局（风险类）**：今晚追加 1 条随机词缀；若核心不失血，明早多一次遗物三选一。
4. **走私商队（经济桥）**：声望与魔力矿双向兑换。
5. **雇佣兵营地（招募类）**：用声望获得随机 4 费或 7 费干员。
6. **废弃军械库（风险类）**：获得建材，但今晚追加随机夜晚词缀。
7. **魔力裂隙（经济/风险类）**：引导获得魔力矿并追加法抗词缀，或用石材封堵换声望。

后续扩充（P2+）：

- **坍塌隧道（空间类）**：永久封死一个刷怪方向，但其余方向今晚出怪 +50%；
- **废弃哨塔（空间类）**：该格建造费用减半；
- **兑换所（经济桥）**：材料 ↔ 声望双向兑换（汇率劣于产出，作为应急阀）。

**当前刷新规则**：每天开始时刷新事件点；第 1 天保底 2 个，此后每天 1-2 个，活跃上限 4 个。落点优先探索前沿的迷雾，而不是地图生成期一次性放置。

### 3.3 商店改进

- **槽位锁定**：每页可锁 1 格未购买干员保留到次日；
- **权重漂移**：随天数推移，出现概率向玩家持有数量最多的 1-2 个盟约轻微倾斜（建议 +15%~25% 相对权重），缓解凑人数的纯随机挫败。

### 3.4 经济桥接

将**魔力矿**升格为跨系统硬通货：高级建筑消耗之外，新增"定向升星"（魔力矿 + 声望直接给指定干员 +1 星，价格高于市场合成期望）与部分事件契约的支付手段。目的：让建筑经济与干员经济在一种稀缺资源上打架。

## 4. 夜晚侧改造

### 4.1 多波结构（最高优先级前置项）

- 每晚 2-3 波，波间 10-15 秒喘息；第 1 天 1 波，随天数递增；
- 目的：让再部署、SP 经济、技能轴、突袭/不屈盟约获得发酵时间；
- 数据层：`wave_templates.json` 的模板改为"波"粒度，夜晚配置引用波列表（或模板内 entries 分组加 `wave_index` + 组间触发条件"上一波清空或超时"）。

### 4.2 夜晚词缀（Night Affix）

每晚随机 0-2 条全局修饰，**清晨随敌情预览一并公示**（公示管线已存在）。词缀独立于模板抽取，与契约可叠加。

当前已落地词缀池为 7 条，见 `data/night_affixes.json`。下表保留为扩展方向，其中“浓雾”尚未实现，且“巫术潮汐”的当前实现仅给敌方法抗 +20。

| 词缀 | 效果 | 检验的构筑 |
|---|---|---|
| 急行军 | 敌人移速 +30%，生命 −20% | 阻挡/减速 |
| 亡语之夜 | 死亡爆炸/分裂类效果增强 50% | 站位、远程 |
| 空袭警报 | 追加一队飞行单位 | 对空火力 |
| 重甲推进 | 披甲单位防御 +30% | 法术/真伤 |
| 巫术潮汐 | 敌方法抗 +20，我方技能 SP 消耗 −10% | 物理轴、技能轴 |
| 西门失守 | 随机一个刷怪点出怪量 ×2，其余 ×0.5 | 防线弹性 |
| 浓雾 | 远程干员射程 −1，近战攻击 +15% | 配队多样性 |
| 嗜血 | 敌人击杀干员后回血，我方击杀声望 +30% | 风险收益 |

实现：词缀作用于波次展开（数量/构成）或全局 buff 管线（数值），与 `_resolve_wave_entry` / 强度系数同层结算。

### 4.3 模板与方差

- 启用闲置的 `enemy_choices` 加权替换，所有模板补 1-2 个可替换槽位（同一模板每局面貌不同）；
- 每档模板补到 6-8 个（词缀上线后，模板本身的方差压力减半，扩充优先级可后移）。

### 4.4 强度系数

新增按天数缩放的全局曲线（数量与数值分开两条），把难度成长从模板硬编码中解放。平衡期只调曲线，不动模板。建议形态：`第 N 晚需求战力 = 基线 × 1.13^N`（系数待跑批校准，见 §7）。

### 4.5 动态出怪口（2026-06-10 重设计）

原"第 4-5 天新增刷怪点"方案被取代，新方向为**模板组化 + 角色分配 + 出怪口层配置化**：

- 模板内部改为若干怪物组，组只声明进攻角色（`main`/`flank`/`any`），不再写死出怪口——"出什么怪"与"从哪出"解耦，口数增减对模板透明；
- 地图生成 5 个候选口（第 1 天起全部可见），每晚活跃口集合由 `ACTIVE_GATES_BY_DAY` 配置表驱动，**每波**主攻口 seeded 重抽，夜内形成方向轮转；
- 全部分配在清晨结算并随敌情预览公示，夜间不重摇——预览即契约。

完整设计与验收见 `docs/superpowers/specs/2026-06-10-dynamic-spawn-gates-design.md`。

### 4.6 敌人扩充

新增原型（每种 1-2 个变体）：

| 原型 | 行为 | 设计目的 |
|---|---|---|
| 战地医师 | 治疗半径内敌人 | 给精准/突袭"斩首"流派存在理由，制造优先级决策 |
| 战鼓手 | 光环加攻速/移速 | 同上 |
| 召唤图腾 | 站桩持续生成小怪 | 必须主动拔点，检验远程/空降 |
| 掠袭者 | **无视核心，直奔最近采集/增益建筑** | 让白天基建必须考虑防守纵深，咬合昼夜两阶段 |
| 疾行者 | 高速低血，绕开重兵路线 | 漏怪检验、防线完整性 |

### 4.7 Boss

- 至少 3 个。现有两个本质都是"双形态数值检验"（一检物理一检法术），第三个直接检验**空间轴**：例如沿途摧毁建筑回血、或分阶段切换进攻路线的 Boss；
- 幕结构（§6）落地后，幕 Boss 在第 3/6 晚提前检验构筑，避免"Boss 内容只在最后一晚"。

## 5. 盟约系统微调

骨架不动，配合上述改动的接口性调整：

- 遗物盟约钥匙件、祭坛加 tag、商店权重漂移均以盟约 id 为索引，`covenant_manager` 暴露查询接口（当前激活档位/层数/差几人）；
- 三合一可行性随天数扩展（§6）后，层数公式才有真实成长空间，届时再做层数收益的平衡 pass；
- 远见（按拥有计算的经济盟约）的"层数 ≥10 解锁卖价折半"在现行体量下不可达，随天数扩展重估或下调阈值。

## 6. 体量扩展：幕结构（3 幕 × 3 天）

**前置条件：§3-§4 的质量改造先行。体量是放大器——放大好决策也放大无聊。** 建议质量改造完成后先在 6 天版本玩测，确认"刚成型就结束"的反馈后再扩。

- **结构**：9 天 = 3 幕 × 3 天，第 3/6/9 晚为幕 Boss；
- **每幕一个难度档 + 专属威胁主题**：第 1 幕基础威胁、第 2 幕支援型敌人登场、第 3 幕拆除与攻城；
- **幕间大节点**：保底稀有/传说遗物 + 新刷怪点 telegraph + 幕间大选择（三选一：定向升星一名干员 / 移除一条词缀 / 一份高级契约）；
- **体量收益**：遗物 8 次三选一；商店 ~45-50 槽使二星稳定可达、三星成为高投入选择，盟约层数轴激活；经济曲线出现中段；
- **时长定位变更需团队拍板**：9 天 ×（白天 1.5-2 分钟 + 夜晚 2-3 分钟）≈ 30-40 分钟/局，偏离文档原定 15-25 分钟。配套缓解：白天操作批量化提效（行动力操作、商店、部署的点击精简）；
- 玩家任意时刻距下一里程碑 ≤2 晚，结构性消灭中段平原。

## 7. 平衡方法论

所有现存数值均为占位（无平衡 pass），填数前先立框架：

### 7.1 锚点：def/res 阶梯

平减法公式下 atk% 是阈值跨越器，先写死各档断点（示例，以现 `enemies.json` 为参考）：

| 档位 | def 区间 | res 区间 | 代表 |
|---|---|---|---|
| early | ≤8 | ≤10 | 士兵、猎狗 |
| mid | 22-36 | 30 | 轻甲卫兵、持盾精锐、术师 |
| late | 36-55 | 35-55 | 重装防御者、萨卡兹大剑手、高阶术师 |
| boss | 55-70 | 40-55 | 两 Boss 一阶段 |

遗物攻击词条与三合一倍率的预算 = "玩家第 N 晚期望攻击应跨越第 N 档断点"。

### 7.2 强度预算

- 基线定义：无遗物、纯 1 星合理配队 → 稳过第 1-2 晚、勉强过第 3 晚、必然过不了第 5 晚；
- 单位预算：每次遗物 pick ≈ 全队有效战力 +10-15%；激活一个 2 人盟约 ≈ 流派内 +25-40%；一次三合一 ≈ 单体 +60%（现 ×1.6 吻合）；
- 词缀 ≈ 当晚难度 +15-25%；契约的奖励预算 ≈ 其难度增量的 1.2-1.5 倍（风险溢价）。

### 7.3 工具

基于现有 `scripts/debug/combat_sandbox.gd` 与 `test_night_template_flow.gd` 搭无头自动战斗跑批：

- 回归矩阵：模板 × 词缀 × 典型构筑（每盟约一套 + 无构筑基线），输出通关余量（核心剩余 HP、超时、击杀率）；
- 验收带：基线队在对应天数模板的通关余量落在目标区间（如第 1 晚余量 60-80%，第 5 晚基线必败、成型构筑余量 20-50%）；
- 每次改数值跑一遍矩阵作为回归保险。

## 8. 落地优先级

### 8.1 P2 任务清单（2026-06-10 重列）

P0/P1 解决的是"系统骨架"：夜晚有了方差，遗物有了结构，事件有了职能。P2 的统一主题是**咬合**——当前商店与盟约、白天经济与夜晚威胁、地图与防线仍各转各的，P2 的每一项都在把其中一对焊起来。建议执行顺序：P2-1 → P2-5 → P2-6 → P2-2 → P2-3 → P2-4（先小后大，寻路/地图改动放最后；其中 P2-4 已于 2026-06-10 重设计并提前开工，见该条目）。

**P2-0 UI 补课清单（2026-06-10 审计，穿插执行）**

P0/P1 的逻辑层基本完整，但部分信息玩家在界面上不可见。原则：**逻辑已生效的系统必须可见**。按严重度：

1. 祭坛灌注的盟约 tag：干员详情面板（`unit_detail_panel.gd` 只读静态配置）与部署卡均看不到灌注结果，需接入 `RunState` 运行时盟约查询；
2. 遗物三选一无槽位来源标注（盟约导向槽/经济槽），玩家无法理解分槽抽取机制；
3. 事件选项 requires 不满足时仅禁用按钮，无"缺什么"的原因提示（`event_panel.gd`）；
4. 波间无倒计时（`WAVE_LULL_SEC` 存在但 HUD 不显示）；
5. 活跃事件数量上限（4）玩家不可感知。

注：夜间词缀显示、预览按波分段两项与 P2-4 直接相关，已并入 P2-4 v1 范围（见设计稿）。

**P2-1 商店锁定槽 + 盟约权重漂移**（商店 ↔ 盟约）
- 锁定槽：每页可锁定 1 个未购买槽位保留到次日重抽；手动刷新（2 声望）清空锁定。
- 权重漂移：第 3 天起，玩家持有人数最多的前 2 个盟约，其成员干员的商店出现权重 ×1.2（占位值）；盟约 presence 用 `RunState.get_unit_covenants`（含祭坛灌注）。
- 触点：`shop_manager.gd`（库存生成/刷新）、`build_panel.gd`（锁定按钮与状态）、`shop_stock_changed` 信号载荷加 `locked`。
- 验收：headless 断言锁定槽跨天保留、刷新清锁、漂移权重表正确。

**P2-2 支援型敌人首批：战地医师 + 战鼓手**（敌人 ↔ 构筑检验）
- 战地医师：光环治疗周围敌人（`support_heal {radius, heal_per_sec}`）；战鼓手：光环加移速/攻速（`support_aura {radius, move_speed_percent, aspd_add}`）。
- 各 1-2 个变体进 mid/late 模板；`wave_manager._key_enemy_score` 给支援类加分使预览优先提示。
- 设计目的：给精准/突袭"斩首"流派制造优先级决策。
- 触点：`enemies.json`、`enemy_actor.gd`（光环行为）、`wave_templates.json`。
- 验收：沙盒断言光环生效、源死亡后效果消失；预览 key_enemies 命中。

**P2-3 掠袭者：威胁经济建筑的敌人**（白天经济 ↔ 夜晚防守）
- 新 `behavior_type: raider`：寻路目标为最近的采集/增益建筑，摧毁后切换下一个，无建筑时转向核心。参考拆除型的建筑攻击逻辑。
- 1-2 个变体（快速低血/中型）进 mid/late 模板。
- 触点：`enemies.json`、`enemy_actor.gd`、`path_service.gd`（以建筑为目标）、`building_manager`。
- 验收：沙盒断言优先打建筑、建筑毁后转向、白天修复流程不受影响。

**P2-4 动态出怪口：模板组化 + 角色分配**（地图/探索 ↔ 防线；2026-06-10 重设计，取代原"第 4 天预告、第 5 天生效"方案）**→ v2 已落地（2026-06-11）**

v1 与 v2 均已实现。具体落地内容：5 个候选出怪口由 `map_generator.gd` 按等弧算法沿边缘放置（`map_generation.json` 新增 `spawn_corner_margin: 3`、`spawn_arc_center_ratio: 0.6`，删除旧 `min_spawn_core_distance`/`min_spawn_distance`）；`night_template_resolver.gd` 新增 `ACTIVE_COUNT_BY_DAY = {1:2, 3:3, 5:4, 7:5}` 日程表与 `resolve_activation_order`/`resolve_active_gates` 两个静态函数；RunState 持有三个一夜覆盖字段（`night_gate_closed_keys`/`night_gate_extra_open_keys`/`night_gate_seals_today`），黎明清零，变更通过 EventBus `night_gate_overrides_changed` 广播；`wave_manager._active_spawn_keys()` 为预览与运行时唯一来源，`get_night_preview` 暴露 `active_gates`；玩家可通过 `day_manager.try_seal_spawn_gate(cell)` 封口（4 石 + 6 AP，最多 1 次/天，不得使活跃口低于 1）；新增事件：塌方契约（动态按活跃口生成选项，消耗 3 魔力矿，封口今晚）和开口赌约（`max_day: 6`，接受追加一个沉默口，收 3 声望 + 2 矿）；UI 层：全部 5 个口以标记显示（活跃亮/沉默暗，封/开角标），封口弹窗可穿透迷雾，夜晚预览添加"今晚活跃口"行，黎明公告新增"战线扩张"提示。新增测试套件 `scripts/debug/test_spawn_gates_v2.gd`（8 个断言场景）。

延期项（显式不在 v2 范围内）：异动警报事件（强制提前开口预警）、口与地形耦合（留待地形包迭代）、`spawn_surge` 词缀语义重做（仍挂账）。

**地形包阶段 A（已落地，2026-06-11）**

highland 地形引擎支持（`TERRAIN_HIGHLAND`：敌人阻挡+不可建造+仅狙击/术师可部署，渲染占位暖黄，`tile_highland.png` 待阶段 B）、人工高台建筑（2木+2石+2AP，500HP，`building_type: block` 出现在阻挡标签页，`ranged_deployable: true`，摧毁时同格干员阵亡）、`reveal_area` 口格不变式修复（出怪口格不再被探索揭开）。调试地图序列化新增 `"highland"` 数组键（缺省 `[]`）。新增测试套件 `scripts/debug/test_highland_platform.gd`。阶段 B（骨架生成器：牌组/车道/长肉/修复/天然高台放置，`tile_highland.png` 美术，沙盒 highland 画笔）待做。

**P2-5 魔力矿经济桥收尾：定向升星**（材料经济 ↔ 干员经济）
- 契约支付侧已落地（祭坛/裂隙/商队），剩"定向升星"：白天对指定干员花魔力矿+声望直接 +1 星（占位：1→2 星 = 3 矿 + 4 声望；2→3 星 = 6 矿 + 8 声望；上限 3 星），定价高于商店合成期望。
- 触点：`RunState`（公开安全的升星入口，复用 `_set_owned_operator_star_no_emit`）、干员详情/出售面板加按钮。
- 验收：headless 断言扣费、升星、满星拒绝、已部署实例的处理。

**P2-6 遗物第二批：盟约钥匙 ×2 + 技能轴词条族**（遗物 ↔ 技能/盟约）
- 每盟约补到 2 件钥匙（首批 1 件已落地）；技能轴补 2-3 件：弹药类技能 +2 发、屏障类效果 +50%、永续类攻速翻倍（首个"开技能全队回 SP"已落地）。
- 纯数据 + 各技能脚本读取点；优先选改动集中在 `unit_skill_behavior` 基类的词条。
- 验收：扩展 `test_relic_draw.gd`。

### 8.2 P3 任务清单

P3 的主题是**扩体量与统一平衡**。两个硬前置：① P2 全部完成并在 6 天版本上玩测，确认玩家普遍反馈"构筑刚成型就结束了"；② 团队拍板接受单局时长从 15-25 分钟变为 30-40 分钟（见"风险与开放问题"第 1 条）。前置不满足时不要启动 P3-1。

**P3-1 幕结构：3 幕 × 3 天（9 天单局）**
- 设计意图见 §6（幕主题、幕间大节点、里程碑密度）。
- 实现要点：`NightTemplateResolver.WAVE_TIERS_BY_DAY` 扩到 9 天并按幕分档；幕 Boss 固定第 3/6/9 晚；`game_controller._on_night_cleared` 的胜利判定从 `day >= 6` 改为数据驱动的总天数；`NightAffixService.AFFIX_COUNT_BY_DAY`、部署上限成长（`RunState.DEPLOY_LIMIT_INCREASE_DAYS` 相关）、商店档位权重随天数曲线同步扩展。
- 幕间大节点：第 3/6 晚结算后插入"幕间选择"流程（保底稀有/传说遗物 + 新刷怪点 telegraph + 三选一大奖励：定向升星 / 移除词缀 / 高级契约）——复用 blessing 流程的 UI 框架。
- 验收：headless 跑通 9 天全流程；各天计划/词缀/商店曲线断言。

**P3-2 第三 Boss（空间检验型）**
- 意图：现有两个 Boss 都是数值检验（一检物理一检法术），第三个检验"建造"这条构筑轴。两个候选方向选其一：沿途摧毁建筑回血（与掠袭者行为共享代码）或第二阶段切换进攻路线（复用动态刷怪点的槽位机制）。
- 触点：`enemies.json`（boss 条目 + phases）、`enemy_actor.gd`、boss 档模板 +1。
- 验收：沙盒断言阶段切换与特殊行为；boss 档模板池 ≥3 后第 6/9 晚重复感显著下降。

**P3-3 波次模板扩充 + `enemy_choices` 启用**
- `wave_manager._pick_enemy_choice` 已实现加权随机替换且与预览一致，但 15 个模板没有一个使用——这是纯数据工作：每档模板补到 6-8 个，每个模板给 1-2 个条目配 `enemy_choices`（同档敌人互换，权重各 1）。
- P2-2/P2-3 的新敌人应进入替换池，让支援型/掠袭者以变体形式出现。
- 验收：同 seed 下预览与实际刷怪一致（既有断言已覆盖）；不同 seed 下同一模板构成可变。

**P3-4 平衡统一 pass：def/res 阶梯定稿 + 自动跑批 + 全局填数**
- 这是全项目数值从"占位"转"定稿"的唯一入口，方法论与验收带见 §7（先锚点、再预算、后填数，不要跳步）。
- 工具：基于 `scripts/debug/combat_sandbox.gd` 扩展无头自动战斗跑批，输出"模板 × 词缀 × 典型构筑（每盟约一套 + 无构筑基线）"矩阵的通关余量（核心剩余 HP / 超时 / 击杀率）。
- 顺序约束：放在 P3 最后做（幕结构和新敌人会改变战力供需，先做会白做）；做完后所有"占位值"标注从本文移除。
- 验收：§7.3 的验收带全部命中；矩阵跑批进 CI 或至少进 `AGENTS.md` 验证命令清单。

### 风险与开放问题

1. **单局时长定位**（15-25 分钟 → 30-40 分钟）需团队明确拍板，P3 幕结构以此为前提；
2. 坚守 3 人"全队伤害均摊"与不屈复活、王命凡形等组合的极端情形，重写遗物池时需排雷；
3. 词缀与契约叠加的难度上限要在跑批矩阵中单独验收（最坏组合不应无解）；
4. 白天操作提效（批量采集、快捷部署）未展开，幕结构落地前需要单独的 UX pass。
