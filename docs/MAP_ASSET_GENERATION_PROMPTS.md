# 地图美术生成：提示词与工作流

> 环境画风：**干净的赛璐璐战场环境**（质感参照明日方舟主线关卡地图）。现有干员/敌人 sprite 是全游戏画风基准——环境是舞台，角色是主角。生成工具 gpt-image-2 网页端（支持参照图），人工生成、AI 实装。
>
> 墙族建筑（`wood_wall` / `artificial_platform`）为程序化绘制（`scripts/building/wall_art.gd`），不走生图；改其美术 = 改该脚本常量并用 `scripts/debug/gen_wall_art_sheet.gd` 出校样。

## 1. 画风五条

写提示词与验收只看这五条：

1. **赛璐璐影调**：每个体块 2-3 阶色阶、影界干脆；不要写实渐变、不要噪点。
2. **统一左上暖光**：阴影是偏冷（蓝紫倾向）的颜色化阴影，不是压暗。
3. **明度层级**：地表最沉最安静 < 地形特征（山/水/高台）< 建筑 < 单位。
4. **棱角化体块、干净剪影**：平顶读成平面；细节集中在受光面与边缘，中心留白可叠单位。
5. **环境色板**：暖绿赭地表 / 青蓝水域 / 暖石色岩体 / 暖木+石建筑。资产画白天标准光，昼夜氛围由代码调色。

## 2. 工作流

1. **每轮附参照图**：`assets/sprites/units/blaze/idle/blaze_idle_000.png` + `assets/sprites/enemies/hound/idle/hound_idle_000.png`；风格锚定稿后再加锚图。
2. **先锚后批**：第 0 轮只生成 3 个资产（见 §4.0），实装后游戏内同框截图验收；过审源图存为 `assets/map/CommandMap/raw/style_anchor_sheet.png`，之后每轮附带。
3. **生成**：复制 §3 提示词头 + 对应批次块。一轮一张源图，存 `assets/map/CommandMap/raw/map_source_sheet_<主题>.png`。写/改任何批次条目前，先想清楚该资产在游戏里如何重复与相邻（成片平铺、多格簇、与邻格拼接），把结论写进条目。
4. **裁切入库**：按批次块内编号顺序裁切，#FF00FF 抠透明。地块 → 缩放 64×64 完整不透明方块，入 `assets/map/CommandMap/<key>.png`；建筑 → 128×128 透明 PNG（主体居中偏下、底部中心锚点、游戏内约 72px 显示），入 `assets/sprites/buildings/<key>.png`。入库前由代码侧后处理：地块边缘中性化保平铺、从大源图裁切平铺变体、特征主体抠出后合成到平地底上；点缀（花/石子）可顺手抠成贴花供合成层撒布。
5. **跑偏就重摇**：影调/光向/色板/抠图任何一项不达标，指出违反画风第几条，直接重新生成；不做后期修补，不用预制纠偏话术。
6. **代码侧分工**：格子外接触阴影、地块间过渡、昼夜调色由引擎合成层负责，不要烘进资产。

## 3. 通用提示词头（每轮复制在最前）

```text
为 Godot 俯视塔防游戏生成素材源图：纯色 #FF00FF 背景，各资产之间留足背景间距、互不接触互不投影；正交俯视，不要透视、等距或六边形。

画风（附图是本游戏的角色 sprite 与已定稿环境资产，生成结果必须与它们同台和谐，环境饱和度低于角色）：干净的赛璐璐战场环境——几何化体块、每个形体 2-3 阶干脆的影调分阶、统一左上暖光、偏冷蓝紫的颜色化阴影、干净剪影；细节集中在受光面与边缘，中心留白；画面透气，层级靠明度对比而不是整体压暗。

禁止：文字、数字、UI 边框/徽章/按钮、角色、敌人、网格线、写实照片质感、PBR 材质、高频噪点、半透明像素、投出资产范围之外的阴影。

为便于后期处理（裁切变体、抠取合成、边缘融合），必须满足：
- 满铺地块（平地、水域、迷雾等会大片相邻拼接的面）：按无缝平铺的连续表面来画——细节均匀散布整格，不要中心构图，不要在四周留出干净的空边圈（多格拼起来会形成格子边框感）；显眼点缀物不贴边不跨边；四边的色调与明度一致。单个地块源尺寸 512 以上。
- 特征格（山/高台/资源点）：主体必须大而满——占格子边长约七到八成、居中不触边，重要物件宁大勿小（最终显示只有 64px，小主体不可读）；主体与地表底之间边界干净、可整体抠出。
- 建筑：只画物体本体，不带地面、草皮、底座，不画投在地面上的影子。
```

## 4. 批次

### 4.0 第 0 轮 · 风格锚（必须先做）

```text
本轮包含 3 个资产，从左到右：

1. tile_plain：平地地块（512x512 不透明方形，最终缩到 64x64）。整张地图最安静的层：暖绿赭地表，明度沉稳不发灰；地表要有细腻的手绘层次——轻微的色相与冷暖变化、柔和的笔触块面，2-4 处碎而淡的异色斑（不要单一大色块），点缀少量两阶草簇/野花/碎石。安静但不单调。
2. tile_mountain：山地阻挡地块（同尺寸）。tile_plain 同款地表底上 2-3 个棱角分明的岩石体块：面片化受光面、干脆影界、偏冷暗部，剪影一眼读出不可通行；冷灰石色，岩石群居中不触边、与地表边界干净。
3. lumber_station：伐木站建筑（最终裁 128x128 透明 PNG）。木料棚+捆扎原木的几何体块组合，暖木色，读得出"资源生产建筑"；只画建筑本体，不带地面草皮底座。
```

### 4.1 地形地块全量（每轮 2-4 个，源 512 → 64x64 不透明方形）

第 1 轮：

```text
1. tile_plain_alt：平地变化格。与已定稿 tile_plain（附图）同族同明度，仅色斑与点缀布局不同，远看仍是平地。
2. tile_hidden：未探索迷雾格。会大片成片拼接：近乎无缝的低对比暗雾面，只有细微云絮明暗变化，无任何独特可辨认物件；不纯黑，不画问号。
3. tile_water：水域阻挡。会大片成片拼接：青蓝水面、2-3 阶水色分层、少量色块化高光均匀散布到四边，不画岸线（水陆交界由代码合成层处理），一眼读出不可通行。
```

第 2 轮：

```text
1. tile_spawn：敌人出生点。地表中心可识别的裂隙/危险符纹，克制的威胁感，不画怪物与传送门。
2. tile_resource_wood：木材资源点。居中的成堆码放原木+粗树桩。
3. tile_resource_stone：石材资源点。居中的整齐石料堆，开采断面外露；人工采集感，与 tile_mountain 的天然山岩明确区分。
4. tile_resource_mana：魔力资源点。居中的晶簇/法力泉，发光克制。
```

第 3 轮：

```text
1. tile_highland：高台地形格。棱角分明的平顶台地：顶面读成干净的平面（不要弧形大明暗的圆丘），崖壁是面片化垂直岩面、背光面偏冷；剪影不对称（可带一侧半高阶地）。暖赭石色，与山的冷灰、平地的绿赭都拉开。顶面中心留白（干员会站上面）。
2. tile_ford：渡口浅滩格。tile_water 同款水面中一条碎石/沙洲浅滩带，浅滩色接近平地表示可通行，不画桥。
```

> 随机事件是悬浮在地块上方的气泡（`map_root_view.gd` 代码绘制），不是地块，没有 tile_event 资产。

> tile_highland 落地前，游戏内由山地贴图加暖色染色占位（`map_root_view.gd`）。

### 4.2 建筑（128x128 透明 PNG，主体居中偏下、底部锚点、72px 显示可读）

第 4 轮：

```text
1. stone_quarry：石矿场。采石台+几块石料，石块剪影清楚。
2. mana_extractor：魔力抽取器。低矮基座围住晶体/抽取柱，发光克制。
3. medical_station：医疗站。治疗帐篷/站台，浅色识别元素，不画红十字按钮。
4. gravity_tower：重力塔。竖向塔芯+石金属底座，不要过高，不画电弧。
5. inspiring_monolith：鼓舞石碑。低矮符文石碑，底座稳重，不要强发光。
6. core_structure：核心（玩家守护目标，立体物而非地块纹理）。中型水晶能量塔/圣物基座：石质底层 + 醒目的核心晶体，体量在本批里最大最庄重，是全图视觉锚点；克制的内蕴光，不要强外发光，不要 UI 感。
```

> core_structure 入 `assets/map/CommandMap/core_structure.png`；实装时核心从地块贴图（tile_core）改为叠在平地上的立体 sprite（`core_view.gd` 渲染），tile_core 随之废弃。

第 5 轮：

```text
1. war_shrine_inactive：战火圣坛·关闭。低矮石质祭坛，中心火熄/余烬，安静。
2. war_shrine_active：战火圣坛·开启。与关闭态结构、尺寸、锚点完全一致，仅中心多克制的实色火焰/能量核。
3. generic_destroyed_building：通用建筑残骸。破碎木板+低矮石块+弯折支架，低矮不挡路，可代表任意建筑损毁态。
```

### 4.3 覆盖层（可选，现由代码绘制，不急）

hover/选中/范围/部署/路线等覆盖层若要资产化：实色主体 + #FF00FF 镂空，运行时引擎统一调透明度；轻薄服务读图，不抢地形与单位。需要时再写具体批次。

## 5. 入库检查

- 文件名 = 资产 key，路径见 §2.4；改 PNG 后跑 `/Applications/Godot.app/Contents/MacOS/Godot --headless --import --path . --quit`，提交 `.png` + `.png.import`。
- 无 #FF00FF 残边、无半透明像素；地块是完整不透明方形。
- 满铺地块（平地/水/迷雾）各自 10×10 平铺无棋盘纹、无接缝、无格子边框感；资源三类在 64px 下不混淆；高台/山/平地三者色相一眼分开。
- 游戏内与干员同框截图，对照 §1 画风五条逐条过。
