# 玩法循环与模块分工

> 本文是了解项目现状的第一入口，描述当前已实现的玩法循环与模块划分。数值以代码/`data` 为准（本文给出的是结构性关键值，细数见对应 JSON 与脚本常量）；平衡数值仍多为占位。

## 一、玩法循环

游戏以“**九天为一个完整周期、分三幕（每幕 3 天，第 3/6/9 晚为幕末 Boss 晚）**”为核心，循环涵盖以下四大环节：

------

### 1. 开局与初始化

- **地图生成：**
  基于随机种子生成 30×30 方块网格，并校验每个出怪口到核心存在可通行路径。默认生成器 `terrain_first`（按高度/湿度场分类的自然地貌，含蜿蜒河流）。
  地图被划分为：
  - 可通行平地（可建造 / 可通行）
  - 不可通行地形（山岳 / 水域）
  - 天然高台（阻挡通行、不可建造、仅狙击/术师可部署）
  - 资源采集点（三种：木材、石材、魔力矿）
  - 出怪口（5 个固定候选，位于地图边缘；每晚按天数只激活其中一部分）
- **迷雾覆盖：**
  核心周围 5×5 区域初始可见，其余区域被黑雾覆盖。
- **核心降落：**
  - 核心建筑（10 HP）落在生成器选定的位置（浮动选址，不一定在正中央）
  - 玩家获得初始建设资源：3 木材、2 石材、1 魔力矿

------

### 2. 白天运营阶段（策略与构筑）

每日资源：

- 行动力：40 点（不跨天累计，每天重置）
- 默认部署上限：5 个；每经过 2 个完整天数 +1（第 3 天 6、第 5 天 7、第 7 天 8、第 9 天 9），遗物可继续提升

#### 行为列表

- **探索（消耗 2 行动力）**
  - 只能选未探索且与已探索区域上下左右相邻的格作为探索中心
  - 消除 3×3 迷雾
  - 可发现资源点或事件点；事件点由每日刷新系统投放到探索前沿
- **基建与生产（建造消耗按建筑配置）**
  - 在资源点建造采集建筑；采集类 / 增益类建筑消耗 2 行动力，木墙消耗 1 行动力，人工高台消耗 2 行动力
  - 已探索资源点可手动采集：消耗 1 行动力，木材/石材/魔力矿资源点均获得 2 个对应资源；每点每天限 1 次，可与采集建筑产出叠加
  - 在空地铺设：路径引导类（木墙）、增益类（医疗 / 重力 / 鼓舞 / 战火圣坛）、远程部署平台（人工高台）
- **随机事件（触发消耗 2 行动力）**
  - 事件点每日刷新：开局保底 2 个，此后每天 1-2 个（活跃上限 4），优先刷在探索前沿的迷雾中
  - 契约事件均为有代价 / 有赌注的交易，前置不足时整体取消并退还行动力。根类约 9 种：黑市交易、战争赌局、走私商队、古代祭坛（魔力矿为同名干员永久灌注盟约 tag）、雇佣兵营地、废弃军械库、魔力裂隙，以及出怪口相关的塌方契约、开口赌约；事件表当前 27 条（含分支结果项）
- **封堵出怪口（消耗 4 石材 + 6 行动力）**
  - 把一个当前活跃的出怪口封住，仅当晚有效；每天最多封 1 个
- **招募与部署（不消耗行动力）**
  - 用声望招募干员实例，加入已拥有干员槽位 / 编队
  - 两段式拖拽部署：从底部待部署区拖到地图格松手锁定落点，再向外拖选朝向松手确认
  - 商店机制：
    - 每天刷新一页，每页 5 个槽位；可花 2 声望刷新
    - 三档价格：2 / 4 / 7 声望
    - 抽取权重按三幕分档（后期高费更易出）：第 1-3 天 65% / 28% / 7%，第 4-6 天 50% / 35% / 15%，第 7-9 天 35% / 38% / 27%
    - 盟约权重漂移：第 3 天起，当前持有人数最多的前 2 个盟约，其成员干员权重 ×1.2
    - 同一页允许重复干员；购买后该槽位锁定，直到刷新或新一天重抽
  - 定向升星：白天可花魔力矿 + 声望直接给指定干员 +1 星（上限 3 星），定价高于商店合成期望
- **建筑修复（不消耗行动力）**
  - 完全损坏：消耗 50% 建造资源修复
  - 未完全损坏：白天自动回满

------

### 3. 夜间战斗阶段（塔防 + 微操）

- **出怪口**
  - 地图有 5 个固定候选口；每晚按天数激活一部分：第 1-3 天 2 口、第 4-5 天 3 口、第 6-7 天 4 口、第 8-9 天 5 口
  - 当晚活跃口集合在清晨随敌情预览公示
- **波次结构**
  - 每晚由按天数解析的多波档位序列组成（第 1 晚单波，第 2-9 晚 2-3 波；幕末第 3/6 晚末波为 Boss，第 9 晚为三 Boss 连战：爱国者 → 奶龙 → 凑凑企鹅）
  - 波间喘息约 12 秒；上一波刷怪完毕后残敌迟迟未清，最多 45 秒强制开下一波（防拖延）
- **夜晚词缀**
  - 词缀池 7 条（全局修饰，如急行军、空袭警报、巫术潮汐）
  - 每晚附加数逐幕加压：第 1-3 天 1 条、第 4-6 天 2 条、第 7-9 天 3 条；受 `min_day` 门控，第 1 晚实际无词缀
  - 清晨随敌情预览公示
- **寻路与进攻**
  - 采用**共享距离场 + 单调下行**（flow field，非每怪 A*）：怪从出怪口沿“到核心的距离场”逐格折线推进，同距并列的格按个体相位在横向上分流——开阔地铺成一面、狭谷收成一股，严格单调必达核心、不踱步
  - 三类 AI：
    - 普通型：沿距离场推进，软避让拦路干员（开阔绕行、窄口走进去接敌）
    - 拆除型：可拆挡路的城墙 / 建筑
    - 飞行型：无视地形，直线飞向核心
  - 路线预览为覆盖面形式（每个出怪口异色），与右侧敌情预览的出怪口编号 + 配色一一对应
- **自动交战**
  - 单位自动攻击；阻挡数决定拦截能力
- **核心扣血**
  - 怪物进入核心后按等级扣血并消失，进入核心的怪物不结算击杀声望
- **微操系统**
  - **技能释放：** SP 满后手动释放
  - **单位详情：** 点击已部署单位显示 HP、SP、属性、技能说明和攻击范围；点其他空格取消选中
  - **时间控制：** 暂停、1x、2x 倍速，暂停时战斗与刷怪队列停止但 UI 可操作
  - **撤退再部署：** 撤退或死亡都让该干员槽位进入再部署 CD，CD 结束后可满血重新部署；同类单位的其他槽位不受影响

------

### 4. 结算与肉鸽阶段

- **战斗结算**
  - 胜利：发放奖励
  - 失败：核心 HP = 0
- **每日遗物**
  - 每晚清场后从 3 件遗物中选 1 件（永久生效）
  - 分槽软导向：盟约导向槽 + 稀有度随机槽 + 经济保底槽
  - 稀有度按天门控：第 1-3 天仅普通；第 4-6 天普通 + 稀有；第 7-9 天稀有 + 传说
  - 遗物效果覆盖职业强化、部署策略、资源经济、建筑协同、核心防线和风险收益
- **周期推进**
  - 共 9 天，分三幕；第 3 / 6 / 9 晚为幕末 Boss 战，第 9 晚通关

------

## 二、玩法模块

------

### 1. 总架构模块

#### 功能

- 核心状态机（阶段流转）
- 全局数据管理（资源 / 声望 / Buff / 天数与幕）
- 项目规范制定（目录 / Git）

#### 关联

- 控制所有模块启停
- 解析 JSON 数据

------

### 2. 地图与关卡模块

#### 功能

- 地图生成（30×30，默认 `terrain_first` 自然地貌生成器）
- 网格状态管理
- 迷雾系统

#### 关联

- 怪物 AI：提供距离场寻路数据
- 基建：验证建造合法性
- 战斗：提供坐标系统

#### 地形类型

- 平地（可建造 / 可通行）
- 资源点（可采集 / 可破坏）
- 不可通行地形（山岳 / 水域）
- 天然高台（阻挡 / 仅远程可部署）
- 出怪口

------

### 3. 战斗与塔防模块

#### 功能

- 干员实例槽位生命周期管理
- 按槽位独立结算的再部署系统
- 伤害 / 治疗计算
- 阻挡与技能系统
- 商店系统 + 定向升星

#### 单位属性

- 生命 / 攻击 / 防御
- 法抗 / 阻挡数
- 攻击范围 / 攻速
- 再部署时间

#### 干员槽位规则

- `unit_id` 表示单位类型和基础数值。
- `operator_key` 表示玩家实际持有的一名干员实例。
- 商店重复购买同一个 `unit_id` 会新增多个不同 `operator_key`。
- 部署、撤退、死亡、再部署 CD 都按 `operator_key` 结算。

#### 单位类型

- 近卫 / 重装 / 狙击 / 术师（4 职业，当前 28 个干员配置）

------

### 4. 怪物与 AI 模块

#### 功能

- 波次控制（按天解析多波模板计划 + 夜晚词缀）
- 寻路 AI（共享距离场 + 单调下行）
- 阻挡 / 仇恨
- Boss 多阶段机制

#### 怪物类型

- 普通型（小兵 / 精英 / 远程，沿距离场推进）
- 拆除型
- 飞行型

当前共 24 种敌人配置。

#### Boss

- 当前 3 个双形态 Boss：奶龙酋长、爱国者、凑凑企鹅
- 第一阶段 HP 归零后进入约 2 秒无敌转阶段，随后切换第二阶段，范围技能变化

------

### 5. 基建模块

#### 功能

- 建筑系统（建造 / 损毁 / 修复）
- 经济系统（行动力 / 建材校验）

#### 建筑类型（共 9 种，详见 `data/buildings.json`）

- **采集类**
  - 伐木站
  - 石矿场
  - 魔力矿场
- **增益类**
  - 医疗站（回血）
  - 重力塔（减速）
  - 鼓舞石碑（攻速）
  - 战火圣坛（攻击加成 + 夜间消耗资源）
- **阻挡类**
  - 木墙（路径引导）
  - 人工高台（狙击 / 术师远程部署平台）

------

### 6. UI 模块

#### 功能

- `CombatHud`（顶部状态、底部待部署区、单位详情、暂停/倍速、今晚敌情预览与波间倒计时）
- `BuildPanel`（建筑/商店复合面板；建筑列表由 `buildings.json` 的 `building_type` 和 `sort_order` 驱动，商店列表由 `ShopManager` 库存驱动）
- `ActionPanel`（白天待机、探索、建造入口和进入夜晚）
- 随机事件 UI
- 肉鸽三选一界面
- 敌情预览：覆盖面路线预览（每个出怪口异色）+ 右侧出怪口编号情报，二者编号与配色对应
- 统一显示文本工具：集中维护职业、阶级、伤害类型、方向、阶段和占位图标的文本映射

------

### 7. 美工与数据模块

#### 功能

- **资产拼装**：素材整合、UI / 动画制作
- **数值配置**：单位表（属性 / 职业）、刷怪表（波次与词缀）、Buff / 事件表
- **灰度测试**：调整难度曲线（按天数全局缩放），保证单局体验

#### 特点

- 数据驱动（JSON）
- 程序只写逻辑，不写数值（平衡数值多为占位，待统一平衡 pass）
