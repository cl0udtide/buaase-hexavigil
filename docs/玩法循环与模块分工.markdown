# 玩法循环与模块分工

## 一、玩法循环

游戏以“单局 15-25 分钟、6 天为一个完整周期”为核心，循环涵盖以下四大环节：

------

### 1. 开局与初始化

- **地图生成：**
  系统基于随机种子生成 30×30 的方块网格地图，并校验刷怪点到核心存在可通行路径。
  地图被划分为：
  - 可通行区域
  - 不可通行障碍
  - 资源采集点（三种：木材、石材、魔力矿）
  - 刷怪点（当前 3 个，位于地图边缘）
- **迷雾覆盖：**
  除中心 5×5 区域可见外，其余区域初始被黑雾覆盖。
- **核心降落：**
  - 核心建筑（10 HP）固定在地图中央
  - 玩家获得初始建设资源：3 木材、2 石材、1 魔力矿

------

### 2. 白天运营阶段（策略与构筑）

每日资源：

- 行动力：30 点（不跨天累计）
- 初始声望：8 点；夜晚击杀敌人获得声望，未击杀而进入核心的敌人不发放声望
- 默认部署上限：4 个；祝福可提升部署上限

#### 行为列表

- **探索（消耗 2 行动力）**
  - 只能选择未探索且与已探索区域上下左右相邻的格子作为探索中心
  - 消除 3×3 迷雾
  - 发现资源点或随机事件（?）；当前正式地图事件点数量为 0
- **基建与生产（建造消耗按建筑配置）**
  - 在资源点建造采集建筑；当前采集建筑和增益建筑消耗 2 行动力，木墙消耗 1 行动力
  - 可在已探索资源点手动采集：消耗 1 行动力，获得 1 个对应资源；每个资源点每天限 1 次，可与采集建筑产出叠加
  - 在空地铺设：
    - 路径引导类建筑（如城墙）
    - 增益类建筑（回血 / 减速）
- **随机事件（当前不额外消耗行动力）**
  - 触发 ? 节点，随机正负效果；当前正式地图默认不生成事件点
- **招募与部署（不消耗行动力）**
  - 使用声望招募干员实例，加入已拥有干员槽位 / 编队槽位
  - 部署指定干员槽位并设定攻击朝向
  - 作战沙盒中采用两段式拖拽验证正式部署体验：
    - 从底部待部署区拖拽干员卡到地图格，松手锁定落点
    - 再从落点向外拖拽选择上下左右朝向，松手确认部署
  - 商店机制：
    - 每天刷新一页，每页 5 个干员槽位
    - 三档价格：1 / 3 / 7 声望
    - 可花 2 声望刷新
    - 固定抽取权重：1 声望一阶 60%，3 声望二阶 30%，7 声望三阶 10%
    - 同一页允许重复干员；购买后该槽位标记为已购买，直到刷新或新一天重抽
  - 同类单位可重复招募，每次招募得到独立槽位
  - 场上单位默认最大部署数量为 4，可通过祝福提升
- **建筑修复（不消耗行动力）**
  - 完全损坏：消耗 50% 建造资源修复
  - 未完全损坏：白天自动回满

------

### 3. 夜间战斗阶段（塔防 + 微操）

- **刷怪点**
  - 每局固定 3 个
  - 每晚根据逻辑生成怪物组合
  - 最后一晚固定 Boss
- **寻路与进攻**
  - 使用 A* 算法
  - 三类 AI：
    - 绕路型
    - 拆除型
    - 无视地形型（飞行）
  - 绕路型默认避开城墙；若核心被城墙完全封闭导致无可用路线，会临时按拆除型路径处理并拆除路上的城墙。
- **自动交战**
  - 单位自动攻击
  - 阻挡数决定拦截能力
- **核心扣血**
  - 怪物进入核心后按等级扣血并消失，进入核心的怪物不结算击杀声望
- **微操系统**
  - **技能释放：** SP 满后手动释放
  - **单位详情：** 点击已部署单位显示 HP、SP、属性、技能说明和攻击范围；点击其他空格取消选中并隐藏范围预览
  - **时间控制：** 作战 UI 提供暂停、1x、2x 倍速，暂停时战斗与刷怪队列停止但 UI 可继续操作
  - **撤退再部署：**
    - 撤退或死亡都会让对应干员槽位进入再部署 CD
    - 该槽位 CD 结束后可满血重新部署
    - 同类单位的其他槽位不受影响

------

### 4. 结算与肉鸽阶段

- **战斗结算**
  - 胜利：发放奖励
  - 失败：核心 HP = 0
- **每日祝福**
  - 三选一 Buff（永久生效）
- **周期推进**
  - 共 6 天
  - 第 6 天 Boss 战

------

## 二、玩法模块

------

### 1. 总架构模块

#### 功能

- 核心状态机（阶段流转）
- 全局数据管理（资源 / 声望 / Buff）
- 项目规范制定（目录 / Git）

#### 关联

- 控制所有模块启停
- 解析 JSON / CSV 数据

------

### 2. 地图与关卡模块

#### 功能

- 地图生成（30×30）
- 网格状态管理
- 迷雾系统

#### 关联

- 怪物 AI：提供寻路数据
- 基建：验证建造合法性
- 战斗：提供坐标系统

#### 地形类型

- 平地（可建造 / 可通行）
- 资源点（可采集 / 可破坏）
- 不可通行地形（山 / 沼泽）
- 刷怪点

------

### 3. 战斗与塔防模块

#### 功能

- 干员实例槽位生命周期管理
- 按槽位独立结算的再部署系统
- 伤害 / 治疗计算
- 阻挡与技能系统
- 商店系统

#### 单位属性

- 生命 / 攻击 / 防御
- 法抗 / 阻挡数
- 攻击范围 / 攻速
- 再部署时间

#### 干员槽位规则

- `unit_id` 表示单位类型和基础数值。
- `operator_key` 表示玩家实际持有的一名干员实例。
- 商店重复购买同一个 `unit_id` 会新增多个不同 `operator_key`。
- 部署、撤退、死亡、再部署 CD 都按 `operator_key` 结算。

#### 单位类型

- 近卫 / 重装 / 射手 / 术士

------

### 4. 怪物与 AI 模块

#### 功能

- 波次控制
- 寻路 AI
- 仇恨系统
- Boss 多阶段机制

#### 怪物类型

- 绕路型（小兵 / 精英 / 远程 / Boss）
- 拆除型
- 飞行型

#### Boss 示例

- 双形态
- 第一阶段 HP 归零后进入 2 秒无敌转阶段，随后切换到第二阶段
- 范围技能变化

------

### 5. 基建模块

#### 功能

- 建筑系统（建造 / 损毁 / 修复）
- 经济系统（行动力校验）

#### 建筑类型

- **采集类**
  - 伐木站
  - 石矿场
  - 魔力矿场
- **增益类**
  - 医疗站（回血）
  - 重力塔（减速）
  - 鼓舞石碑（攻速）
  - 战火圣坛（攻击加成 + 消耗资源）

------

### 6. UI 模块

#### 功能

- `CombatHud`（顶部状态、底部待部署区、单位详情、暂停/倍速）
- `BuildPanel`（建筑/商店复合面板；建筑列表由 `buildings.json` 的 `building_type` 和 `sort_order` 驱动，商店列表由 `ShopManager` 库存驱动）
- `ActionPanel`（白天待机、探索、建造入口和进入夜晚）
- 随机事件 UI
- 肉鸽三选一界面
- 统一显示文本工具（规划）：集中维护职业、阶级、伤害类型、方向、阶段和占位图标文本映射，避免多个 UI 脚本重复维护同一套显示规则

------

### 7. 美工与数据模块

#### 功能

- **资产拼装**
  - 素材整合
  - UI / 动画制作
- **数值配置**
  - 单位表（属性 / 职业）
  - 刷怪表（波次设计）
  - Buff 表
- **灰度测试**
  - 调整难度曲线
  - 保证 20 分钟体验

#### 特点

- 数据驱动（JSON）
- 程序只写逻辑，不写数值
