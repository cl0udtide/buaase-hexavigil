# 角色与敌人美术生成：提示词与工作流

> 现有干员/敌人 sprite（高饱和赛璐璐、Q 版、干净细描边）是全游戏画风基准，不重绘——环境美术反过来以它们为参照（见 `MAP_ASSET_GENERATION_PROMPTS.md`）。本文档只在新增角色/敌人（通常是新 Boss）时用：找一张外观参照图，用 gpt-image-2（网页端，支持参照图）重画成同族 sprite，人工生成、AI 实装。
>
> 不管 UI 头像图标（`UI_ASSET_GENERATION_PROMPTS.md`）、地图建筑（`MAP_ASSET_GENERATION_PROMPTS.md`）、战斗特效（`EFFECT_ASSET_GENERATION_PROMPTS.md`）。

## 1. 画风五条

写提示词与验收只看这五条：

1. **同族基准**：Q 版二到三头身、高饱和赛璐璐影调（2-3 阶硬影界）、深色干净细描边、材质分区清晰（金属/布料/皮肤/甲壳）、克制的局部高光。生成时必须附 1-2 张现有 sprite 锚画风。
2. **角色是主角**：饱和度与对比明显高于环境；每个角色 1-2 个强色重音作识别点；发光只作局部识别，不抢范围/特效层级。
3. **视角与朝向**：俯视正交或轻微三分之二俯视，默认朝右（左向由代码 flip_h）；不要正面立绘、半身头像、卡牌插画、海报构图、强透视。
4. **轮廓优先**：70px 下先读出职业/敌人类型、体型、武器或核心识别点。
5. **干净抠图**：纯色 #FF00FF 背景出图，最终抠成 128×128 透明 PNG；主体完整居中略偏下、脚底中心为锚点、四周留透明边距；边缘干净无洋红残边。普通单位可见约 64-82px，Boss 可放大到约 86-96px。

## 2. 工作流

1. **附参照图**：每次附 1-2 张现有 sprite 锚画风（`assets/sprites/units/blaze/idle/blaze_idle_000.png`、`assets/sprites/enemies/hound/idle/hound_idle_000.png`），再附该角色的外观参照图。
2. **生成**：复制 §3 提示词头 + §4 角色描述，一次出一张源图，存 `assets/sprites/<units|enemies>/raw/<名>_redraw_source.png`。一个两形态 Boss 也只出一张 sprite，形态差异由配置与特效表现，不为各形态分别出图。
3. **裁切入库**：#FF00FF 抠透明 → 128×128 透明 PNG，存 `assets/sprites/<units|enemies>/<名>/idle/<名>_idle_000.png`（data 里按这个名字引用）。
4. **导入提交**：`/Applications/Godot.app/Contents/MacOS/Godot --headless --import --path . --quit`，提交 `.png` 与 `.png.import`。
5. **跑偏就重摇**：影调/朝向/抠图任一不达标，指出违反画风第几条，直接重生成；不做后期修补，不用纠偏话术。

## 3. 通用提示词头（每次复制在最前）

```text
请基于上传的参照图，重画为 Godot 俯视塔防游戏的地图 sprite：纯色 #FF00FF 背景，最终裁成 128×128 透明 PNG。另附的本游戏现有 sprite 是画风基准，结果必须与之完全同族——高饱和赛璐璐、Q 版二到三头身、2-3 阶硬影界、深色干净细描边、材质分区清晰、克制的局部高光。

俯视正交或轻微三分之二俯视，默认朝右；主体完整居中略偏下、脚底中心为锚点、四周留透明边距；70px 下先读出类型、体型、武器与核心识别点。

参照图只继承身份、轮廓、配色和关键识别点；丢掉原图的背景、UI 框、文字、卡牌构图和读不出的高频细节。

禁止：背景地块、大底座或投在地面的影子、UI 边框/徽章/头像框、文字数字水印、技能弹道/爆炸/范围圈（属特效）、写实照片、PBR 3D、霓虹强发光。
```

## 4. 角色描述（接在提示词头后）

```text
本次重画 <一句话定位，例：一只穿绒甲的企鹅形 Boss>：
- 类型：<干员职业 / 敌人类型：普通、Boss、飞行、远程、拆除……>
- 轮廓与识别点：<体型、站姿、头部、武器、护甲、族群——70px 下要一眼认出的 3-5 点>
- 主色 / 重音色：<主色> / <功能或危险重音色>
- 必须保留 / 可简化 / 需强化：<从参照图继承的> / <高频纹理、巨大披风等> / <最重要的轮廓或颜色>

飞行单位可略悬浮，但不画巨大地面影子。
```

无参照图、纯文字新建时，去掉提示词头里"参照图只继承……"那句，其余照常。

### 参照示例：奶龙酋长（已入库，取材网络梗）

两形态 Boss，整局共用这一张 sprite。重画正文（接 §3 头之后）：

```text
本次重画奶龙酋长——一条圆胖、厚鳞、表情呆萌的奶黄色小龙 Boss：
- 类型：敌人 Boss
- 轮廓与识别点：矮胖龙身、厚重鳞甲、短钝犄角、圆滚剪影；体量比普通敌人大但不压满画布。
- 主色 / 重音色：奶黄鳞甲 / 橙红鳍背与爪。
- 必须保留：呆萌脸、厚鳞、圆胖比例；可简化：背景与高频鳞片纹理；需强化：70px 下圆胖剪影与奶黄主色。
```

## 5. 入库检查

- PNG 为 128×128、透明、无 #FF00FF 残边；`git diff --check` 无空白错误。
- 在暗色平地、山地、水域、路径线和范围覆盖层上预览，70px 下轮廓清楚；Boss 不遮挡邻格单位与攻击范围提示。
- 与现有干员同框截图，对照 §1 画风五条逐条过。
